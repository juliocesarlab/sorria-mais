O diretório lp/ está vazio. Vamos construir do zero um site institucional premium para a clínica odontológica Sorria Mais, com presença em Uberaba e Uberlândia, posicionada como referência em implantes dentários e facetas em resina.

A entrega precisa parecer trabalho de agência high-end (não template, não IA genérica, não Lovable, não "startup"). Stack: HTML + CSS + JS vanilla, sem React/Next/Tailwind/Bootstrap. Bibliotecas leves de animação são permitidas se realmente agregarem (GSAP + ScrollTrigger e Lenis para scroll suave).

Identidade visual:

Fundo branco, primária #530946, muito espaço em branco, tipografia editorial sofisticada, linhas finas e divisórias sutis.
Sem gradientes pesados, sem sombras fortes, sem grids repetitivos de "4 cards iguais".
Prioridades técnicas: SEO local impecável (Schema Dentist/LocalBusiness com 2 unidades), HTML semântico, performance (Lighthouse alto), JS mínimo, conteúdo indexável imediatamente.

Design system (em styles.css)
Tokens (CSS custom properties no :root):

Cores: --ink: #1a1014, --brand: #530946, --brand-soft: #f6eef3, --brand-line: #ead8e3, --bg: #ffffff, --muted: #6b6168.
Tipografia: sans arredondada e moderna para títulos e corpo — opção principal Nunito (ou alternativa Quicksand / Mulish) via Google Fonts display=swap com preconnect. Forma orgânica e amigável das letras (terminações suaves, "o" e "a" arredondados) traz acolhimento sem perder sofisticação. Apenas dois pesos (ex: 400 e 700) para reduzir payload. Nada de serifa editorial nem fonte manuscrita.
Escala tipográfica fluida com clamp() (h1 ~ clamp(2.6rem, 6vw, 5.5rem)).
Espaçamento: sistema de 8px, com seções respirando (padding-block: clamp(6rem, 12vw, 10rem)).
Linhas/divisórias: 1px solid var(--brand-line) para detalhes editoriais.
Princípios de layout:

Grid editorial assimétrico (CSS Grid com grid-template-columns de 12 colunas, mas usando spans irregulares).
Mistura de alinhamento: títulos à esquerda, números à direita, imagens fora da grade quebrando margem.
Sem cards-com-sombra. Profundidade vem de tipografia, espaço e linhas finas.

Animações (script.js)
Lenis para scroll suave global (instanciar com lerp: 0.1, smoothWheel: true).
GSAP + ScrollTrigger para reveals: títulos entram com y: 40, opacity: 0 → 1, stagger sutil em parágrafos. Discreto e curto (duração 0.8s, ease expo.out).
Header: classe .is-scrolled adicionada após 24px de scroll, transição CSS.
Carrossel: rAF custom (não GSAP) para performance.
Menu mobile: classe .menu-open no <body>, transição via CSS.
prefers-reduced-motion: reduce desabilita Lenis e reveals.
IntersectionObserver como fallback se GSAP não carregar (degradação graceful).
SEO técnico
Apenas um H1 (hero). Cada <section> usa <h2>, subseções <h3>.
meta description ~155 caracteres focada em conversão local.
JSON-LD validado mentalmente contra schema.org/Dentist.
alt descritivo em todas as imagens (ex: "Sala de atendimento da Sorria Mais Uberaba").
loading="lazy" em tudo abaixo da hero; hero usa fetchpriority="high".
robots.txt permitindo tudo + apontando sitemap.
sitemap.xml listando home + âncoras principais.
Nenhum keyword stuffing — palavras-chave aparecem naturalmente em h2, parágrafos e schema.
Fontes via Google Fonts com display=swap + preconnect (sem bloqueio de render).
CSS inline crítico opcional (avaliar no fim) — provavelmente não necessário se mantivermos styles.css enxuto.
Verificação
Abrir index.html no navegador (file:// ou python -m http.server) e validar:
Header se transforma ao rolar; menu mobile abre/fecha sem travar.
Carrossel de depoimentos tem fade lateral e pausa ao hover.
Reveals aparecem suaves, sem "pop" ou jitter.
Seções de unidades mostram claramente os dois endereços e CTAs.
Lighthouse (Chrome DevTools, modo mobile): mirar Performance ≥ 90, SEO 100, Acessibilidade ≥ 95, Best Practices ≥ 95.
Validar JSON-LD em https://validator.schema.org/ (colar o <script type="application/ld+json">).
Testar prefers-reduced-motion (DevTools → Rendering) — animações devem desaparecer.
Verificar com leitor de tela básico (NVDA/VoiceOver) que landmarks (header, main, nav, footer) estão corretos e que apenas um h1 existe.
Mobile: testar em viewport 375px — composição editorial deve continuar elegante (não quebrar em cards genéricos).
